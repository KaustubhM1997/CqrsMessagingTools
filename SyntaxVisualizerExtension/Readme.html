The Roslyn Syntax Visualizer is a Visual Studio Extension that will enable inspection of 'live' Roslyn SyntaxTrees for any C# or VB code file that is open inside the Visual Studio IDE when the Roslyn Language Service is present (e.g. when running or debugging other Roslyn extensions).

To use the visualizer, you will need to build the SyntaxVisualizerExtension project. After building, the visualizer extension will be installed and available for all instances of Visual Studio that are started with the Roslyn Language Service. For example, all of the Roslyn code issue and code refactoring samples run in instances of Visual Studio with the Roslyn Language Service - so the visualizer will be available when you are running or debugging any code issue or code refactoring samples.

You can also test the visualizer by setting the SyntaxVisualizerExtension project as the startup project and running (CTRL + F5) or debugging (F5) it.

Here is a step by step outline of what to do:

  • Set the SyntaxVisualizerExtension project as startup project.
  • Build and run this project (CTRL + F5 or F5).
  • This will bring up a new instance of Visual Studio running with the Roslyn Language Service.
  • In this Visual Studio instance, open the visualizer window by clicking on View -> Other Windows -> Roslyn Syntax Visualizer.
  • Open any C# or VB code file. This can either be a standalone code file or a code file that is part of some project.
  • When the code file is opened, the visualizer will display the Roslyn SyntaxTree corresponding to this code file.
  • You can click on / select any SyntaxNode, SyntaxToken or SyntaxTrivia in the visualizer. When you do this, the source text corresponding to the clicked item will also be selected in the code editor.
  • You can also position the cursor / select text anywhere in the code editor. When you do this, the SyntaxNode / SyntaxToken / SyntaxTrivia that best matches the selected source text will be highlighted in the visualizer.
  • You can also inspect the Properties for the currently selected SyntaxNode / SyntaxToken / SyntaxTrivia in the PropertyGrid that is part of the visualizer's user interface.
  • In Visual Studio Premium and Visual Studio Ultimate, you can right click on any SyntaxNode / SyntaxToken / SyntaxTrivia in the visualizer and click on Directed Syntax Graph. This will display a graphical representation of the sub-tree rooted at the selected SyntaxNode / SyntaxToken / SyntaxTrivia. This feature is not available in Visual Studio Professional.
  • You can close the visualizer window at any time if you do not wish to have it running.
  • You can also disable / uninistall the visualizer using Tools -> Extension Manager.
